# NL Housing Fixtures

Static MVP fixtures for the NL housing case. These are the app-facing inputs
used by the FastAPI engine boundary and React cockpit.

Files:

- `candidates.json`
- `evidence.json`
- `support_groups.json`
- `structures.json`
- `personal_profiles.json`
- `market_anchors.json`
- `factor_research.json`

`factor_research.json` is generated by `scripts/build_nl_housing_app_research_fixture.py`
from `work/nl_housing_factor_research/factor_tree.json` plus the tool-fetched web
source registry saved at `work/nl_housing_factor_research/web_sources_v2.json`.
It is a compact mixed-source UI fixture: factor tree nodes, candidate metric lines,
source links, provenance tags, and rollups. It contains no API key.

`factor_research.json` also carries `probability_scoring`, factor/source rollups,
and per-line `tagged_metrics` so the UI can show a single probability percent.
Already happened / already-in-force items are `100`; model estimates start at `80`;
analysis/news claims at `60`; forecasts at `55`; policy targets/proposals at `50`;
risk scenarios at `35`; and rumor/unverified claims at `15`, with small source/framing
adjustments in the fixture builder.

`evidence.json` is generated by `app/build_fixtures.py` from canonical Trace claims.
Each evidence object now has top-level `evidence_status`, `probability_pct`,
`is_already_real`, and rationale fields. They are intentionally not embedded inside
`inferences`, so the deterministic engine loader remains unchanged.

Only nested source objects extracted from raw Cala responses may carry
`source_origin: "cala"` or `source:cala`. Factor nodes use
`derived_from_source_origins` so the UI can highlight direct Cala sources without
mislabeling web/tool evidence as Cala.
Dutch-language web sources carry `language: "nl"` and still use `source:web`.

`personal_profiles.json` contains demo user-input profiles. `quoted_mortgage_rate_pct`
is a user/bank-quote input for Personal Fit; it is not market evidence.
